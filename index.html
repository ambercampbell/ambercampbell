  • Menlo Innovations

    Software Engineering Intern | Ann Arbor, MI | June 2023 - September 2023

    At Menlo, I worked on a web application designed to help healthcare professionals review prescriptions for Direct Oral Anticoagulants (DOACs). It identifies patients on DOACs and applies logic rules to assess prescriptions. The back-end utilizes a daily cron job to generate alerts by pulling and parsing patient data from EHR systems. To comply with HIPAA regulations, it stores only FHIR IDs, fetching patient data as needed. The app is built using HTML, CSS, JavaScript, Python, Django, PostgreSQL, and AWS, and is optimized for deployment within Epic Hyperspace without a complex front-end framework.

    The application addresses prescription errors, which occur in 10-15% of DOAC prescriptions, by providing nurses and pharmacists with an efficient tool to review and correct errors, rather than having to manually review all prescriptions. The dashboard displays alerts associated with specific patient prescriptions, offering detailed patient information and allowing users to take action, such as snoozing or resolving alerts, resulting in intervention in 79% of cases. While similar tools exist in Epic’s EHR, this application was developed as a stand-alone using FHIR interoperability standards, allowing seamless integration with Epic and other EHR systems for broader hospital use.

    The main tasks I worked on during this internship was developing the core logic for identifying prescriptions to flag and what kind of flag to associate with such prescriptions. I also worked on logic that involved caching generated alerts and using this cache to only display unresolved alerts and only re-examine prescriptions if something in the patient’s data has changed. Throughout this internship, I had the opportunity to collaborate closely with a multidisciplinary team, including physicians, project managers, QA specialists, and fellow developers, through pair programming and team discussions. This environment allowed me to contribute to the development of a product that directly benefits both patients and healthcare providers.

To ensure reliability, we incorporated crash-resilient features, including optimized disk I/O operations and secure handling of untrusted inputs. This project deepened my understanding of the challenges involved in building scalable, high-performance, and reliable systems, while enhancing my ability to design solutions for real-world concurrency and fault tolerance.

Memory Manager

Course Project: Advanced Operating Systems

I co-designed and implemented a pager for managing virtual address spaces within an operating system kernel. My work focused on key memory management functions, including address space allocation, memory and swap file management, and the creation of system calls to handle the creation, duplication, and destruction of address spaces, facilitating smooth interaction between user applications and the pager. I developed an interrupt handler to efficiently manage page faults, ensuring that pages were correctly loaded from swap or file storage into physical memory. By supporting both swap-backed and file-backed virtual pages, I optimized memory usage, enhanced system performance, and ensured process isolation.
